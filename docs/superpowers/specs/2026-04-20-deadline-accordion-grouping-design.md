# 데드라인 테이블 아코디언 그룹핑 — 설계 문서

> 작성일: 2026-04-20 | 관련: 컴플라이언스 페이지 UX 개선

---

## 배경

현재 기한초과/임박 데드라인 테이블이 건별로 flat하게 나열되어, 같은 근로자가 여러 줄 반복됩니다. 근로자별로 묶어서 "이 사람이 몇 건 초과/임박"을 한눈에 보여주는 아코디언 그룹핑으로 변경합니다.

---

## 설계

### 아코디언 구조

```
▼ Nguyen Van Minh — 기한초과 2건
    | 설명              | 기한       | 상태   | 처리 |
    | 고용보험 취득신고   | 2025-07-24 | 기한초과 | 완료 |
    | 산재보험 취득신고   | 2025-07-24 | 기한초과 | 완료 |
▶ Ahmad Rizki — 기한초과 4건
▶ Tran Van Duc — 기한초과 1건
```

### 헤더 정보

`{workerName} — {상태라벨} {N}건`

- 기한초과 테이블: `Nguyen Van Minh — 기한초과 2건`
- 임박 테이블: `Nguyen Van Minh — 임박 3건`
- ID(`workerId`)는 헤더에 표시하지 않음 (내부 DB ID, 사용자에게 무의미)

### 기본 상태

- 첫 번째 그룹만 펼침 (가장 심각한 근로자 자동 노출)
- 나머지는 접힌 상태
- 클릭으로 접기/펼치기 토글

### 정렬

- **기한초과 테이블**: 초과 건수 많은 순 (가장 심각한 근로자가 위)
- **임박 테이블**: 가장 급한 dueDate가 가까운 근로자 순

### 펼친 내용

기존 테이블 행과 동일: 설명, 기한, 상태(StatusBadge), 완료 버튼(onComplete 있을 때)

## 변경 범위

### DeadlineTable 컴포넌트 수정

| 파일 | 변경 |
|------|------|
| `components/compliance/deadline-table.tsx` | flat 렌더링 → 아코디언 그룹핑. workerId로 그룹화 → 정렬 → 아코디언 렌더링 |

### 내부 구현

1. **그룹화**: `items`를 `workerId`로 그룹화하여 `Map<number, { workerName: string, deadlines: ComplianceDeadlineResponse[] }>` 생성
2. **정렬**: 그룹을 건수 내림차순 정렬 (기한초과), 또는 최소 dueDate 오름차순 (임박)
3. **아코디언 상태**: `expandedWorkers: Set<number>` — 첫 번째 그룹의 workerId를 기본값으로
4. **렌더링**: 각 그룹별로 헤더(클릭 가능) + 조건부 테이블 행

### 정렬 기준 구분

`DeadlineTable`은 `title` prop으로 기한초과/임박을 구분합니다. 정렬 로직에 새 prop을 추가하는 대신, 기존 prop으로 판단:
- 모든 데드라인의 status가 `OVERDUE`면 → 건수 내림차순
- 그 외 → 최소 dueDate 오름차순

### 외부 인터페이스

`DeadlineTableProps` 변경 없음. 기존 소비자(compliance 페이지, 대시보드, 근로자 상세) 호환 유지.

단, 근로자 상세 페이지(`/workers/[id]`)에서는 한 명의 근로자 데드라인만 표시하므로 그룹이 1개 → 아코디언 없이 기존처럼 flat 렌더링. **데드라인이 단일 근로자일 때는 아코디언 스킵**.

### 페이지네이션과의 관계

현재 compliance 페이지는 외부 페이지네이션(`usePaginatedOverdueDeadlines`)을 사용합니다. 페이지네이션된 결과를 그룹핑하면 한 근로자의 데드라인이 페이지에 걸쳐 나뉠 수 있습니다.

**해결**: 아코디언 그룹핑은 현재 페이지의 `items` 기준으로만 적용. 페이지를 넘기면 같은 근로자가 다시 나올 수 있지만, 이는 현재 데이터 규모(회사당 수십 건)에서 실질적 문제가 아님. 서버 사이드 그룹핑은 향후 과제.

## 범위 외

- 대시보드 요약 데드라인 — `limit` prop 사용 시 기존 flat 유지
